Config.xml

Config.xml是全局配置文件，控制着cordova应用行为的很多特性，这个不分平台的XML文件基于W3C的Packaged Web Apps (Widgets) 规范编写的，被扩展来表述CordovaAPI的核心功能，插件，平台特有设置。

在使用Cordova命令行创建的项目，这个文件在项目目录的顶层

注意： 3.3.1-0.2.0之前的版本，这个文件在app/www/config.xml目录，目前也支持文件存在这个目录下。

widget（插件）

config.xml文档的根元素。

属性（类型） 描述
id(string) 必须项
指定应用程序的反向域标识符，和表示在大版本/小版本/补丁等构建版本下的完整版本号
version(string) 必须项
表示在大版本/小版本/补丁等构建版本的完整版本号
android-versionCode(string) Android
可选安卓应用的版本号，为应用设置版本号，查看Android平台指南了解如何修改这个属性
ios-CFBundleVersion(string) iOS
可选iOS应用版本号。查看iOS版本号获取更多细节
osx-CFBundleVersion(string) OSX
可选OS X版本号。查看OS X版本号获取更多细节
windows-packageVersion(string) Windows
可选Windows版本号。查看Windows版本号获取更多细节
packageName(string) 默认值：Cordova.Example
Windows
Windows应用的包名
xmlns(string) 必须项
config.xml文档的命名空间
xmlns:cdv(string) 必须项
前置命名空间

engine（引擎）

指定在准备过程中要恢复的平台的详细信息

属性（类型） 描述
name(string) 必须项
要恢复的平台名
spec(string) 必须项
要恢复的平台的详细信息。这个的值可以是一个major.minor.patch版本号，一个包含这个平台的路径或者是一个git仓库链接。此信息将用于检索从NPM、本地目录或git仓库恢复的平台代码。更多信息查看Platform Spec
