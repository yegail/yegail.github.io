fastlane是iOS和Android应用自动打包部署最简单的工具，能处理所有c重复乏味的工作，像获取应用截图，处理代码签名，发布应用等。

fastlane

在工程目录下创建一个“Fastfile”文件，文件中定义了beta和App Store发布的任务。

定义了两个lane，一个是开发的，一个是发布到App Store的。使用命令fastlane beta提交beta版到TestFlight，fastlane appstore发布应用到App Store。

为啥使用fastlane

  1. 节约每次打包发布到App Store或beta版测试服务器的时间；
  2. 集成了所有的的工具和服务（目前有170种操作）；
  3. fastlane源码100%开源；
  4. 能在很短时间内启动开发助手；
  5. 在设备上自动运行你的应用；
  6. 集成了所有主要的CI系统；
  7. 支持iOS，Mac和Android应用；
  8. 支持扩展和自定义fastlane，以满足你的需求，且不会依赖任何其他库；
  9. 不需要记住其他命令，仅仅是fastlane就够了；
  10. 可以在任何计算机上部署，包括CI服务器。

开始

确定已经安装了最新的Xcode command line tools：

xcode-select --install

选择安装方式

（1） Homebrew:

brew cask install fastlane

（2） Installer Script: 下载fastlane，然后双击install script 或者世界在terminal中运行install script。

（3） rubygems:

sudo gem install fastlane -NV

命令行cd到你的iOS或Android工程目录下，然后输入

fastlane init

fastlane会自动查明你的工程，问你缺少的信息。想要获取关于设置和运行的更多细节，请看入门指南：

  • fastlane iOS入门指南
  • fastlane Android入门指南

使用Fabric app

如果你不熟悉命令行，更喜欢使用图形界面的工具，可以下载Fabric app，使用built-in fastlne 按钮。

问题和支持

  • 查看docs.fastlane.tools
  • 查看README页面
  • 搜索已有的GitHub issues

当以上都没能解决你的问题时，请在GitHub上提交一个issue，请提供您的设置信息，特别是in particular命令的输出。

系统需求

目前，fastlane官方支持MacOS，在其他系统上还是有限制，而且没有测试过。fastlne使用的系统API可能没有被其他平台实现，例如：我们使用’Ruby fork’方法进行过程管理，在Windows上就不可用。
